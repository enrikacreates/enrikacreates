Mobile-conversion effort (by stack)

  • Vite SPA + Supabase → Capacitor wrap: ~3–10 days, ~90–95% code reuse. Swap web camera/file APIs for Capacitor plugins.
  • Next + Supabase → Capacitor (after client-render restructure): ~1–3 weeks, ~75–85% reuse. Risk of Apple 4.2 "thin wrapper" rejection if you just wrap the hosted site.
  • Next + Payload + Neon → new Expo client vs the Payload API: ~3–8 weeks. Backend reused ~100%, frontend rebuilt (~0%).
  • Already Expo/RN (ReachOut): zero conversion — just eas build.
  • The bottleneck is developer time, not compute. Backend & its cost are unchanged. Builds run on EAS free tier or locally on a Mac.

Recommendations

  • SPAs (SignatureStyle, StoryKeepr, The Daily Story, etc.): "build web first, Capacitor-wrap when demand appears" is cheap and correct (~days each).
  • Create Space v2 (top app, not built yet, wants web + mobile): build greenfield in Expo + Expo Router + Supabase — one codebase ships web + iOS + Android, zero conversion tax. Use it as the proof-of-concept for the Expo-default decision. (Expo and Vite are alternative frontends, not a combo — Expo Router covers web too.)
  • Vision Map (near-future, hardest path): plan the "Expo client against the Payload API" approach early; keep all data access via the API.
  • Store It: same Payload→Expo path, but most mature so the effort is most justified.
  • Cheapest demand-test: ship a PWA (installable, $0, no rebuild) before committing to Capacitor or Expo.

Cost reference

  • Supabase: Pro = $25/mo per org (covers one Micro instance). All Supabase apps share ONE project (schema per app) = flat $25/mo. Each additional Supabase project ≈ +$10/mo.
  • Neon: the 5 Payload apps each have their own DB (already isolated). Free tier + usage-based scaling.
  • Mobile (per app): Apple Developer $99/yr, Google Play $25 one-time. Expo SDK free; EAS has a free build tier (or build locally for $0). Going mobile adds no backend/DB cost.
